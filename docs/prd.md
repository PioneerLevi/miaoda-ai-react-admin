# 需求文档

## 1. 应用概述

**应用名称**：Leavis个人主页

**应用描述**：一个简约清爽的个人主页网站，用于展示个人信息和作品，并提供数字分身聊天功能，方便访客了解和联系。

## 2. 用户与使用场景

**目标用户**：
- 朋友
- 潜在合作伙伴
- 面试官

**核心使用场景**：
- 访客浏览个人信息，了解Leavis的背景和兴趣
- 访客通过数字分身聊天区提问，获取关于Leavis的信息
- 访客在移动端访问主页

## 3. 页面结构与功能说明

### 页面结构

```
个人主页
├── 个人信息展示区
└── 数字分身聊天区
```

### 3.1 个人信息展示区

**功能说明**：
- 展示头像
- 展示名字：Leavis
- 展示一句话介绍：一个正在学习用 AI 写程序的java开发人员
- 展示个人信息：
  - 我现在主要在做：整理自己的作品和技术路线
  - 我的兴趣：ai应用，java开发，钓鱼，骑行
  - 我一个比较有记忆点的特点：喜欢抽象的概念，内向

### 3.2 数字分身聊天区

**功能说明**：
- 访客在输入框输入问题
- 访客点击发送按钮提交问题
- 数字分身根据预设信息回答问题
- 聊天记录展示在对话区域

**数字分身预设信息**：
- 职业或身份：Java开发
- 最近在做的事：搭自己的个人主页，整理做过的项目和技术路线
- 擅长或关心的方向：内容表达、AI 应用、知识整理
- 常见问题及回答方向：
  - 你现在在做什么？
  - 你有哪些作品？
  - 怎么联系你？

## 4. 业务规则与逻辑

### 4.1 数字分身回答逻辑

- 数字分身基于预设信息生成回答
- 当访客提问与预设信息相关时，数字分身提供对应回答
- 当访客提问超出预设信息范围时，数字分身提示「这个问题我暂时无法回答，你可以尝试问我其他问题」

### 4.2 移动端适配

- 页面布局在移动端自动调整，确保信息清晰可读
- 聊天区输入框和按钮在移动端易于操作

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 访客输入空白问题 | 提示「请输入问题」 |
| 访客连续快速发送多条问题 | 按顺序依次回答 |
| 数字分身无法回答问题 | 提示「这个问题我暂时无法回答，你可以尝试问我其他问题」 |

## 6. 验收标准

1. 访客打开个人主页，看到头像、名字、一句话介绍和个人信息展示区
2. 访客在数字分身聊天区输入问题「你现在在做什么？」
3. 数字分身回答「我正在搭自己的个人主页，整理做过的项目和技术路线」
4. 访客在移动端打开主页，页面布局清晰，信息完整展示
